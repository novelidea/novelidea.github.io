Shuffling Algorithm

简介

洗牌算法是一个比较经典的算法，主要讨论如何随机生成一副牌。第一次遇见是2013年春腾讯一面的时候，当时妥妥跪了，题目都没听懂，第二次遇见是大三上系统结构课，最后讲到了洗牌算法。比较经典详细的算法解释参见维基百科。这里主要讨论一种流行的简化版本。

设计实现

主要思路是从扑克的最后一张开始，假设下标为 i，然后从在区间[0, i - 1]中生成随机数j，交换下标为i，j所对应的元素。如此从最后一张循环操作到第二张（到第一张时可以跳出循环）。

C++代码如下:

# include <iostream>

define CARDNUM 52

using namespace std;

class Shuffling{
public:
int shuffle(){
int cards = new int[CARDNUM];
int index = 0;
for(int i = 1; i <= 13; i ++){
for(int j = 0; j < 4; j ++){
cards[index ++] = i;
}
}
for(int i = CARDNUM - 1; i >= 1; i –){
int j = rand() % i;
swap(cards, i, j);
}
return cards;
}

void swap(int arr[], int a, int b){
    int tmp = arr[a];
    arr[a] = arr[b];
    arr[b] = tmp;
}

};

int main() {
Shuffling action = Shuffling();
int* a = action.shuffle();
for(int i = 0; i < CARDNUM; i ++){
cout<<a[i]<<endl;
}
return 0;
}

Java代码如下：

public int[] shuffle(int max){
        Random rand = new Random();
        int[] cards = new int[max];
        for(int i = 0; i < max; i ++){
            cards[i] = i;
        }
        for(int i = max - 1; i > 0; i –){
            int exchange = rand.nextInt(i);
            int tmp = cards[i];
            cards[i] = cards[exchange];
            cards[exchange] = tmp;
        }
        return cards;
    }

其中，这里需要注意的是，Java中如果要生成区间[min, max]之间的随机数，则是：

rand.nextInt(max - min + 1) + min;

写在岁末年初（2015）

转眼又到岁末年初，真没想到一个系列一写就是8年。

这一年发生了很多事，五味杂成，以至于想找今年的主基调却找不到，因为感觉很多经历都是极端化的存在，从交大毕业结束本科学习又步入南加大开始新的学业，和老同学分别又认识了新的朋友，看上去这一年都在不断地经历着改变，但是在这其中却仍有很多东西一成不变，曾和朋友开玩笑说，只是换个地方写代码罢了，比如还在上学做作业考试、还在上数据库研究关系模型背各种范式、还在刷leetcode、还是宅不喜欢外出……

一年确实能改变很多，非常清楚的记得14年的最后一天的晚上6点多列了一个计划，共9条，有的超额完成，有的没做，有的做了但没做好。今天我也列了个单子，只有3条，或许这样就可以集中精力将每个做好吧。

2015年，很长，因为感觉毕设已经很遥远，但是想想也很短，也就36个10天，半年前签证被check还记忆犹新。

那就先像大妈一样来唠唠这12个月的家常吧。

1月，应该主要是把毕设的理论部分处理完了。毕设应该算我本科四年中比较遗憾的事，系里分配的方向不是很喜欢，甚至是有点排斥的，但是由于懒惰、嫌麻烦，所以就安慰自己说“锻炼如何做好自己不喜欢的事情”，也因为没有了兴趣，所以就打算速战速决、匆匆做完了老师所要求的部分。现在想想，如果当初搞个爬虫、或者做个图像处理、或者做个实践项目，应该可以做得更有意思。

2月，周围申请同学的offer陆续下来了，我却杳无音信，所以就开始准备找工作，寒假在家刷leetcode，锻炼算法能力是一方面，另一方面也是想让自己忙起来，转移注意力。

3月，开学，还没有offer，真的开始找工作了，投了两个公司。那段日子想想真有意思，正如一句话说的，怀念的往往是过的不太好的日子，每天来回站近4个小时的公交，就这样持续面了一周，终于在3月20号的下午4点多接到了实习邀请 －收拾东西，三天后开始实习。所以我三天后梦幻般地到了广州，第一次踏出学校，非常感激公司以及那么一群人，耐心地给我指导、给我关照。非常怀念15年3月那个只要穿短袖的广州。另外，3月21日凌晨4点，也就是接到实习邀请的十几个小时后我收到了第一个offer，Rutgers University，在实习期间也收到了SBU的录取。

4月，广州的实习结束，在广州玩了一天，第一次现场听交响乐、逛了广州塔、吃了广州菜，还有就是晚上12点骑车单车在路上找旅馆。回到西安后似乎没有经历更多的事情，应该主要在写毕设论文吧。

5月，首先应该是毕业杯吧，终于打上了班级的比赛，投了三个球，一个45度3分，一个0度底线，一个中投，但是，一个都没进……不过这次比赛感觉打得都很“随意”，因为毕业杯也许是很多人和做了四年的对手、队友间的最后一场比赛；还有羽毛球的毕业杯，在比赛中学会了不太标准的扣杀。5月19号，拿到了USC的summer录取，然后又转了个fall，改签了机票和签证。

6月，申请结束，毕设结束，拍毕业照，第一次喝吐，毕业不是很开心。签证被check。

7月，15年的低谷吧，不过是时隔10年在老家过的暑假，10年里第一次过着与山为邻与鹤为伍的日子，还买了钓鱼竿钓鱼。

8月，走完亲戚踏上了新的征程，快速融入了一个完全陌生的环境，然后上课。初到美国适应非常的快，甚至在飞机上就把时差给倒完了，学习也没有传说中的累——可能主要原因是选了两门比较水的课吧。

9月，已经忘了经历了什么了，只记得月底搬家整理东西的时候翻出一张博物馆的门票，上面的日期是9.05。额，另外9月好像还会按时完成算法课的作业。

10月，参加了career fair，同届的很多同学真的有备而来啊，穿的正装、简历也很漂亮，排着facebook、 google、amazon等的超长的队伍，想想自己前一天晚上才决定去看看修改简历、当天睡到老晚，真是惭愧。所以，fall career fair就打了个酱油，看了看行情：cs的竞争蛮激烈的，而且ios开发不是很好找工作。

11月，shooting、月考、开始复习final，boring。

12月，做了个independent project，同学过来玩，好几拨，有见到的，也有没见到的。搬了个家。最后的10天基本是在新家中宅、看电影、综艺。

匆匆几百字就写完了2015……

回顾2015这一年，能够用来回忆、聊的事其实挺多的，虽然大部分都是十分钟内能讲完的小事。

15年有意思的事情有很多，比如版权，刚到美国的时候，百度音乐还是能用的，但是现在就用不了了；之前在西安的时候基本上每个月都感冒一次，但是来了美国的半年中，一次感冒都没有；12月一个朋友过来玩已经认不出我了，原因是4月份的时候我是圆脸，现在已经瘦成长脸了。

15年尝试了很多之前想做但是没有做或者没做成功的事情，比如组建了discussion group，承蒙兄弟们瞧得起，加入这个group、拉各种牛入伙，关于这个group，其实大二的时候在班里办过一个，当时只是去刷九度online judge题，即使简单，然而也只持续了两周就散伙了，而这次是从成立开始一直坚持到了final的前两周，总结经验教训便是：志同道合的兄弟真的很重要！在15年的尾巴上我也学会了滑滑板，溜冰吉他是我初中时候最像做的事情，但是由于种种原因一直没有去尝试，最接近的一次是大二的时候买了双旱冰鞋，结果在楼道里磕到了消防箱子之后就将它束之高阁了；15年也尝试了一次喝醉，感觉真不好，曾经和朋友说要将第一次喝醉留给失恋，呵呵……

15年有很多值得留恋的事情，比如毕业卖书，买了三年的旧书终于到了自己在东八楼底下抛头露面，曾经有无数个设想，但到头来还是和往届一样，打牌吆喝吃东西，曾经的学弟在学完了高数线代大物数据结构概率论嵌入式后终于要和西交说再见了；怀念卖完书后的通宵，唱了歌拍了视频，那应该是第二次通宵，第一次是高考结束。

15年像一首歌，一首节奏忽快忽慢的歌，有过烦心的时候，在交大的操场上戴着耳机跑步或者是坐在操场旁边的台阶上静静地听着歌、想着事情；有伤感的时候，比如毕业的那段日子；有像被催眠了一样，日历像被风吹了一样，哗哗哗地快速地翻过了8月9月10月11月12月。有时真的感慨来了美国后突然感觉时间过的好快，不是时间不够用的快，而是感觉什么事情都没做一个月又结束了。现在想想一大原因可能是睡懒觉，由于这学期不需要起早上课，所有也就没有定过闹钟，每天都是9点10点自然醒的节奏。这个假期小伙伴过来玩聊起了这个话题，他们说一般都是定8点前的闹钟，听完这句话我脑海中就飘过了一句话，“loser，比你优秀的人还比你努力！”，所以，新的一年里必须要践行的一条就是再也不能睡到10点了。

15年，难得的有欢笑也有泪水的一年，在接下来的日子里注定会被经常提起的、注定不平凡的一年，今天就要被贴上封条。

2016年，曾经是那么地陌生、那么地遥远，现在，它的画册即将打开，将由我在接下来的366天内将它的空白填满。

对新的一年有很多的憧憬，也有担忧，但是我坚信大部分的日子都像加州的天气一样，阳光明媚。

WatchOS学习笔记

1.UIImage

设置图片的时候尽量使用setImageNamed，而不是setImageData和setImage，主要原因是第一种方法提前将图片加载到watchKit App Target中去了，而后两种方法的图片在watchKit Extension中，运行时需将图片资源加载到watchKit App Target中，会造成传输中的延迟，影响用户体验。但是在一开始导入图片的时候，如果选择setImageNamed方法，则需要将图片同时加到watchKit App Target和watchKit Extension中，不同于后两种方法，只需要将图片直接加到watchKit Extension即可。这里是用空间换取时间。

1.block

 

block定义：

(returnType)(^blockName)(parameterTypes) = ^(parameters) {

statements

};

该block类型为：(returnType)(^)()

定义block快捷键：inline+enter

定义类型为block的属性语法：

@property (nonatomic, strong) void(^myBlock)(NSString *str) ;

注意点1:由于block是c语言的语法，所以在写block参数属性的时候不同于一般oc定义语法；

注意点2:block名称（myBlock）一定要放在(^)中，不能写成void(^)(NSString *str) myBlock（但是在方法中可以这样定义）;

@protocol XPFBlogViewDelegate <NSObject>
