为何Vertex要定义成静态类？

答：这里可以理解为类方法，假设类A含有内部类B，则将B声明为静态类，即可将B看成A的类方法，生成类B只需要用类A名就可以了，而不需要生成一个类A，即A.B b = new A.B();相反，如果B不是静态类，生成类B则需要先生成类A，即A.B b = new A().new B()。如果该内部类定义为私有，则只能在外部类内访问。

top k问题

简介

top k问题是指给定元素集合，找出前k大的元素，或者找出出现频率前k大的元素。这个问题在算法课上经常讨论，但是却从未实现过，现在就来实现一下改算法。

top k问题暴力求解方法当然是对所有元素的比较依据元素进行排序，然后选出前k大的元素，这里的时间复杂度是O(nlg(n))，空间复杂度是O(1)。但是这个效率不高，改进主要有两种方法，一种是利用堆，一种是利用快排的辅助函数quick search算法。

实现

首先，我们用第一种方法来实现一下，以找出文中出现频率最高的单词为例。首先，我们需要遍历文本，构造｛单词：频率｝的哈希表，这里的时间复杂度是O(1)，空间复杂度是O(n)。然后我们再遍历一遍哈希表，利用小顶堆找出出现频率钱k大的单词，该时间复杂度是O(nlog(k))，空间复杂度是O(k)（关于堆操作的时间空间复杂度在这就不再赘述了）。综上，利用堆来处理topK问题的时间复杂度是O(nlog(k))，空间复杂度是O(n)。

图的遍历分析

图的存储

如果只是粗略地讲图的遍历，即DFS和BFS是完全不用讲这一个点，但是，本着讲（装）透（逼）的原则，还是要简要介绍一下图在计算机中的存储方式，因为在后面的时间复杂度分析中会用到。

图的存储基本有两种，邻接矩阵（Adjacency Matrix）和邻接表（Adjacency List）。首先来说邻接矩阵，顾名思义，邻接矩阵是一个n*n矩阵，其中n是指图中节点数目，横纵坐标对应相应的节点，对于普通图来说，如果某两个节点中有边相连，则将这两点在邻接矩阵中对应的元素赋值为1，反之则为0；对于加权图来说，如果某两点间有边相连，则将邻接矩阵中对应的元素赋值为该权值，反之则为＋Infinity。

邻接矩阵在处理密集图比较好，但是对于稀疏图来说就会存在空间利用率不足的弊端，这就要用到邻接表。邻接表是一种数组与链表相对结合的方式，数组中存放的是图中所有的节点（node），然后每个node又是一个链表的头节点，每个链表中的节点是图中所有与改链表头节点相邻的节点，如果是有向图，则是由头节点指向的节点。可见，邻接表的空间复杂度是和图中节点数和边数成正比的，不像邻接矩阵中的n*n是固定的，所以在处理稀疏图时首选邻接表。

总结一下，图的两种存储的空间复杂度分别为（图中有n个节点，m条边）：

邻接矩阵：O(n * n)

邻接表：O(m + n)

图的遍历

BFS

图的遍历分为深度优先（DFS）和广度优先（BFS）。首先来讲广度优先，广度优先是一层一层的遍历图，由于本篇重在分析，所以略过具体的过程描述，伪代码如下。

set start vertex to visited

load it into queue

while queue not empty

    for each edge incident to vertex

         if its not visited

             load into queue

             mark vertex</pre>

现在我们来分析一下它的时间复杂度，简单起见，我们假设图是用邻接表存储的。假设用集合{v1, v2, v3, …, vn}来表示途中的n个节点，则执行时间可以简单描述为：

v1 + (从v1出去的边数) ＋ v2 ＋ (从v2出去的边数) ＋ v3 ＋ … ＋ vn ＋ (从vn出去的边数)

变换为：

(v1 + v2 + … + vn) + {(从v1出去的边数) + (从v2出去的边数) + … + (从vn出去的边数) }

前一部分是O(n)，后一部分是O(m)，所以，综上，用邻接表存储的图的BFS时间复杂度是O(m + n)。

DFS

深度优先，深度优先就是一条路走到底，然后再走另外一条。同样略过具体实现细节，类比于BFS，DFS也是将所有点和边都走一边，只不过点边的访问顺序不一样，所以，时间复杂度同样为O(m + n)。

空间复杂度

这不同于上文所说的图的存储的空间复杂度，而是图在遍历过程中额外的空间开销。在图的遍历中，为了避免因为图中有环而对同一点重复访问造成死循环，所以引入一个矩阵，来标志某点是否已经被访问，该矩阵长度为n，所以DFS和BFS的空间复杂度就为O(n)。

Java中ArrayList排序

基础

在Java中，如果需要将一个arrayList排序，我们需要利用Collections类。首先先游离一下主题，先简要介绍一下Collections，Collections不同于Collection，后者（java.util.Collection）是一个集合接口，它提供的是对集合对象进行基本操作的通用接口方法，而前者（java.util.Collections）是一个包装类，它包含有各种有关集合操作的静态多态方法，此类不能被实例化，就像一个工具类，服务于Java的Collection框架。

在本文中，我们讨论ArrayList排序，用到的就是Collections的sort方法，下面的例子简单实现了对ArrayList<Integer> 和ArrayList<String>的升序排序和降序排序。

进阶

下面我们主要讨论如何对对象的某个属性进行排序，我们不能直接将对象实例名字作为传入sort函数，这样程序会由于sort函数没有排序比较的依据而报错。我们可以有两种选择，如果只需要根据单一实例变量进行排序，我们只需要让需要比较的类继承Comparable，然后在该类中重写Comparable的comparaTo方法，例子如下：

        return this.age;

        int compareAge = ((Student)comparestu).getAge();
        return this.age - compareAge;
    }
}

区间重叠问题

问题描述

给定n个区间，找出重叠数量最多所对应的点。形象化描述是有4个飞机的schedule:[1,2], [3,4], [2,5], [3,6]，空中最多有多少架飞机。这里答案是3，因为在时刻3和4，空中飞机的数量最多，为3。为了简单起见，有两个假设：

假设1: 区间起点和终点都是整数；

假设2:假设同一时刻有一个飞机降落，另外一架飞机起飞，则先降落后起飞。

暴力解法

首先想到的是greedy algorithm，进行三层遍历循环。先将所有区间按照起始数值排序，然后依次遍历区间数组，在每次遍历某个区间时，又遍历该区间的每个数值，将该数值和区间数组中的剩下的区间进行比较，看是否重合。该算法在OJ上能通过87%的case，但是当测试集变大之后就会超时。算法实现如下：

/**
 * Definition of Interval:
 * public classs Interval {
 *     int start, end;
 *     Interval(int start, int end) {
 *         this.start = start;
 *         this.end = end;
 *     }
 */

class Solution {
    /**
     * @param intervals: An interval array
     * @return: Count of airplanes are in the sky.
     */
    public int countOfAirplanes(List<Interval> airplanes){
        if(airplanes.size() == 0) return 0;
        sortList(airplanes);
        int maxCount = 1;
        while (airplanes.size() > 1){
            Interval head = airplanes.get(0);
            for (int i = head.start; i < head.end; i++){
                int count = 1;
                for (int j = 1; j < airplanes.size(); j++){
                    Interval time = airplanes.get(j);
                    if (time.start <= i && time.end > i){
                        count++;
                    } else if (time.end <= i){
                        airplanes.remove(j);
                    }
                }
                maxCount = Math.max(count, maxCount);
            }
            airplanes.remove(0);
        }
        return maxCount;
    }
    //sort a Interval list by the start point(ascending order)
    private void sortList(List<Interval> list){
        for (int i = 0; i < list.size() - 1; i++){
            for (int j = i + 1; j < list.size(); j++){
                if (list.get(i).start > list.get(j).start){
                    Interval tmp = list.get(i);
                    list.set(i, list.get(j));
                    list.set(j, tmp);
                }
            }
        }
    }
    /*
    TEST CASE:
        [1,1]
        [1,2],[2,3]
    */
}

KMP(Knuth–Morris–Pratt) Algorithm

简介

KMP(Knuth-Morris-Pratt)是较为高级的字符串查找算法，是三个人名字的缩写，主要实现判断一个字符串A是否包含另一个字符串B，暴力解法是遍历循环，时间复杂度是O(mn)，其中m，n分别是字符串AB的长度。

分析

由于暴力解法中没有充分利用上一次遍历查找失败的有用信息，所以时间复杂度很高，而KMP算法主要就是解决这一问题，主要思路如下：

字符串1为：ABCDABCEDEAB

字符串2为：ABCDAB

暴力遍历解法在匹配到字符串2中E时报告匹配错误，然后重新从字符串1中第一个B的位置重新开始匹配，但是我们发现，下一次匹配开始的位置其实可在字符串1下标为4的位置，所以在每次匹配失败后会存在一次跳跃，来剪掉那些不可能匹配成功的“枝”。根据KMP算法，这个跳跃值是（本次已经匹配的字符串长度 － 此时未匹配字符所对应的partiallyMatchValue）。partiallyMatchValue可通过查找partially Match Table得到，该table是表征的是一个｛字符：整数｝的关系，字符是所有待查找字符串中的字符（这里是字符串2），此table通过如下关系得到：

假设我们现在要求字符串2中第二个B所对应的整数值，则首先得到在字符串1中以B结尾的字串为：

字符串3：ABCDAB

计算改字符串3的所有prefix组合：｛A, AB, ABC, ABCD,ABCDA};

计算改字符串3的所有postfix组合：{B, AB, DAB, CDAB, BCDAB}.

然后，字符串3的prefix和postfix交集为{AB}，交集中的长度最大值为2，故partially Match Table中第二个B对应值为2，遍历字符串2可以得到一张长度为字符串2长度的表。

实现

# include <iostream>

using namespace std;

class StringSearch{
public:
bool stringSearch(string str, string target){
int* table = partialMatchTable(target);
int matchedNum = 0;
for(int i = 0; i < str.length();){
if(matchedNum == target.length()){
delete[] table;
return true;
}else if(str[i] == target[matchedNum]){
matchedNum ++;
i++;
}else{
i += (matchedNum - table[matchedNum]);
matchedNum = 0;
}
}
delete[] table;
return false;
}

int* partialMatchTable(string s){
    int len = s.length();
    if(len &lt; 1) return NULL;
    int* table = new int[len];
    memset(table, 0, len);
    for(int i = 2; i &lt;= len; i ++){
        table[i - 1] = longestCommonStrOfPrefixAndPostfix(s.substr(0, i));
    }
    return table;
}

int longestCommonStrOfPrefixAndPostfix(string s){
    int len = s.length();
    if(len &lt; 1) return 0;
    for(int i = 1; i &lt; len; i ++){
        string prefix = s.substr(0, len - i);
        string postfix = s.substr(i, len - i);
        if(prefix == postfix){
            return prefix.length();
        }
    }
    return 0;
}

};

int main() {
StringSearch action = StringSearch();
string s = “abababcdabdab”;
string target = “abcdabd”;
cout<<action.stringSearch(s, target)<<endl;
return 0;
}
