KMP(Knuth–Morris–Pratt) Algorithm

  • Algorithm

简介

KMP(Knuth-Morris-Pratt)是较为高级的字符串查找算法，是三个人名字的缩写，主要实现判断一个字符串A是否包含另一个字符串B，暴力解法是遍历循环，时间复杂度是O(mn)，其中m，n分别是字符串AB的长度。

分析

由于暴力解法中没有充分利用上一次遍历查找失败的有用信息，所以时间复杂度很高，而KMP算法主要就是解决这一问题，主要思路如下：

字符串1为：ABCDABCEDEAB

字符串2为：ABCDAB

暴力遍历解法在匹配到字符串2中E时报告匹配错误，然后重新从字符串1中第一个B的位置重新开始匹配，但是我们发现，下一次匹配开始的位置其实可在字符串1下标为4的位置，所以在每次匹配失败后会存在一次跳跃，来剪掉那些不可能匹配成功的“枝”。根据KMP算法，这个跳跃值是（本次已经匹配的字符串长度 － 此时未匹配字符所对应的partiallyMatchValue）。partiallyMatchValue可通过查找partially Match Table得到，该table是表征的是一个｛字符：整数｝的关系，字符是所有待查找字符串中的字符（这里是字符串2），此table通过如下关系得到：

假设我们现在要求字符串2中第二个B所对应的整数值，则首先得到在字符串1中以B结尾的字串为：

字符串3：ABCDAB

计算改字符串3的所有prefix组合：｛A, AB, ABC, ABCD,ABCDA};

计算改字符串3的所有postfix组合：{B, AB, DAB, CDAB, BCDAB}.

然后，字符串3的prefix和postfix交集为{AB}，交集中的长度最大值为2，故partially Match Table中第二个B对应值为2，遍历字符串2可以得到一张长度为字符串2长度的表。

实现

# include <iostream>

using namespace std;

class StringSearch{
public:
bool stringSearch(string str, string target){
int* table = partialMatchTable(target);
int matchedNum = 0;
for(int i = 0; i < str.length();){
if(matchedNum == target.length()){
delete[] table;
return true;
}else if(str[i] == target[matchedNum]){
matchedNum ++;
i++;
}else{
i += (matchedNum - table[matchedNum]);
matchedNum = 0;
}
}
delete[] table;
return false;
}

int* partialMatchTable(string s){
    int len = s.length();
    if(len &lt; 1) return NULL;
    int* table = new int[len];
    memset(table, 0, len);
    for(int i = 2; i &lt;= len; i ++){
        table[i - 1] = longestCommonStrOfPrefixAndPostfix(s.substr(0, i));
    }
    return table;
}

int longestCommonStrOfPrefixAndPostfix(string s){
    int len = s.length();
    if(len &lt; 1) return 0;
    for(int i = 1; i &lt; len; i ++){
        string prefix = s.substr(0, len - i);
        string postfix = s.substr(i, len - i);
        if(prefix == postfix){
            return prefix.length();
        }
    }
    return 0;
}

};

int main() {
StringSearch action = StringSearch();
string s = “abababcdabdab”;
string target = “abcdabd”;
cout<<action.stringSearch(s, target)<<endl;
return 0;
}
